================================================================
  群晖 Docker 部署步骤
================================================================

【前置】
  - 群晖已安装 Docker 套件
  - 群晖已开启 SSH（控制面板 → 终端机和 SNMP → 终端机）
  - 域名 www.nas7.top DDNS 已解析到公网 IP

【文件】
  部署包: invoice-deploy.tar.gz（4.6MB，含代码 + 1475 条业务数据）
  打包命令: tar -czf invoice-deploy.tar.gz invoice-deploy/

【部署步骤】

1. 上传文件到群晖
   File Station → /volume1/docker/ → 上传 invoice-deploy.tar.gz → 解压

2. SSH 登录群晖
   ssh swingtt@www.nas7.top

3. 创建挂载目录
   sudo mkdir -p /volume1/docker/invoice/pgdata
   sudo mkdir -p /volume1/docker/invoice/uploads

4. 构建镜像
   cd /volume1/docker/invoice-deploy/
   sudo docker-compose build --no-cache

5. 启动容器
   sudo docker-compose up -d

6. 验证启动
   sudo docker-compose ps              # 两个容器都应为 Up
   sudo docker-compose logs invoice    # 看到 gunicorn worker 启动即正常

7. 验证数据
   sudo docker exec invoice-pg psql -U invoice -c "SELECT COUNT(*) FROM invoices;"
   # 应返回 76

8. 防火墙放行 1135 端口
   群晖控制面板 → 安全性 → 防火墙 → 新增规则: 端口 1135 TCP

9. 路由器端口转发
   外部 1135 → 群晖IP 1135 (TCP)

10. 浏览器访问
    http://www.nas7.top:1135
    admin / admin123

【HTTPS 配置（可选）】
  群晖控制面板 → 登录门户 → 高级 → 反向代理:
    来源: HTTPS www.nas7.top 端口: 1135
    目的地: HTTP localhost 端口: 5001
    证书: 选择已有证书
  同时修改 docker-compose.yml:
    ports: "1135:5000" → "127.0.0.1:5001:5000"

【Dockerfile 加速说明】
  - apt: mirrors.tuna.tsinghua.edu.cn
  - Node.js: npmmirror.com/mirrors/node/v18.20.8
  - pip: pypi.tuna.tsinghua.edu.cn
  - npm: registry.npmmirror.com

【常用维护命令】

  # 更新代码后重建（不用 --no-cache，更快）
  sudo docker-compose build
  sudo docker-compose up -d

  # 查看日志
  sudo docker-compose logs -f invoice

  # 进入容器
  sudo docker exec -it flask-invoice sh

  # 备份数据库
  sudo docker exec invoice-pg pg_dump -U invoice > backup_$(date +%Y%m%d).sql

  # 停止/启动
  sudo docker-compose down
  sudo docker-compose up -d

  # 完全重置（清空数据重新导入）
  sudo docker-compose down
  sudo rm -rf /volume1/docker/invoice/pgdata
  sudo mkdir -p /volume1/docker/invoice/pgdata
  sudo docker-compose up -d
