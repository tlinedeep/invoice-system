# 物资点收领用平台 v0.40 — 群晖全新部署步骤

> **部署目标**：群晖 DS920，域名 http://www.tjghaz.com:1135  
> **数据库**：PostgreSQL 15（Docker 容器内）  
> **账户**：swingtt / swing208blue（管理员），admin 已禁用  

---

## 一、准备工作

### 1.1 上传部署包

通过群晖 File Station 或 scp 将 `invoice-deploy.tar.gz` 上传到群晖：

```bash
# 本地电脑上执行（替换为你的群晖 IP）
scp invoice-deploy.tar.gz swingtt@www.tjghaz.com:/volume1/docker/
```

### 1.2 SSH 登录群晖

```bash
ssh swingtt@www.tjghaz.com
```

输入密码后进入群晖命令行。

---

## 二、解压部署包

```bash
cd /volume1/docker/
mkdir -p invoice-deploy
tar -xzf invoice-deploy.tar.gz -C invoice-deploy
cd invoice-deploy
```

---

## 三、创建持久化目录

```bash
sudo mkdir -p pgdata uploads
```

- `pgdata/` — PostgreSQL 数据库文件（数据持久化，删除即丢失数据）
- `uploads/` — 发票上传文件和导出 Excel

---

## 四、一键部署

```bash
bash deploy.sh
```

脚本会自动执行：
1. 构建 Docker 镜像（国内镜像加速：阿里云 apt + 华为云 PyPI + npmmirror）
2. 启动 PostgreSQL + Flask 容器
3. 等待数据库就绪
4. 验证服务状态

---

## 五、验证部署

### 5.1 查看容器状态

```bash
sudo docker-compose ps
```

两个容器都应为 `Up` 状态：
- `invoice-pg` — PostgreSQL 15
- `flask-invoice` — Flask 应用

### 5.2 查看启动日志

```bash
sudo docker-compose logs invoice
```

应看到：
```
[OK] 数据库初始化成功
==================================================
  物资点收领用平台
  启动地址: http://localhost:5000
==================================================
```

### 5.3 验证数据库

```bash
# 验证仓库数据（14个分类含关键词）
sudo docker exec invoice-pg psql -U invoice -c "SELECT code, name FROM warehouses;"

# 验证用户
sudo docker exec invoice-pg psql -U invoice -c "SELECT username, role, enabled FROM users;"
```

结果应看到：
- 14 个仓库（01-钢材 ~ 14-杂品）
- swingtt（admin, enabled）
- admin（admin, disabled）

### 5.4 浏览器访问

打开 http://www.tjghaz.com:1135

## 七、后续升级

以后升级只需两步：

### 7.1 上传新包

将新的 `invoice-deploy.tar.gz` 上传到群晖 `/volume1/docker/`（覆盖旧包）。

### 7.2 SSH 执行升级

```bash
ssh swingtt@www.tjghaz.com
cd /volume1/docker/invoice-deploy/
bash update.sh
```

`update.sh` 会自动完成：
1. 解压新代码（不覆盖 `pgdata/` 和 `uploads/`）
2. 重建 Docker 镜像
3. 重启容器
4. 验证服务状态

> 数据完全保留，不受影响。

---

### 查看日志

```bash
sudo docker-compose logs -f invoice
```

### 重启服务

```bash
sudo docker-compose restart invoice
```

### 更新代码（保留数据）

将新版本 `invoice-deploy.tar.gz` 放到 `invoice-deploy/` 目录下，然后：

```bash
cd /volume1/docker/invoice-deploy/
# 备份配置
cp .secret_key .secret_key.bak 2>/dev/null; true

# 解压新代码（跳过 pgdata/ uploads/）
tar -xzf invoice-deploy-v0.40-fresh.tar.gz --skip-old-files

# 恢复密钥
cp .secret_key.bak .secret_key 2>/dev/null; true

# 重建并重启
sudo docker-compose build --no-cache
sudo docker-compose up -d
```

### 备份数据库

```bash
sudo docker exec invoice-pg pg_dump -U invoice > /volume1/docker/backup_$(date +%Y%m%d).sql
```

### 完全重置（清空所有数据）

```bash
sudo docker-compose down -v
sudo rm -rf /volume1/docker/invoice-deploy/pgdata
# 然后重新执行 deploy.sh
```

## 八、注意事项

1. **域名解析**：确保 `www.tjghaz.com` 已 DDNS 解析到群晖公网 IP
2. **端口冲突**：1135 不能被其他服务占用
3. **群晖防火墙**：如果外部无法访问，检查群晖防火墙和路由器转发
4. **HTTPS**：如需 HTTPS，在群晖控制面板设置反向代理：
   - 来源：HTTPS www.tjghaz.com 端口 443
   - 目的地：HTTP localhost 端口 5000
5. **admin 账户已禁用**：安装后 admin 账号是禁用状态，只能使用 swingtt 登录
6. **仅仓库数据**：点收单、领用单、工程、人员、供应商全部为空，需手动录入
